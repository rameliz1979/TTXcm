Integridad

Verificacion mediante SHA-256

Procedimiento inicial para confirmar si el recurso tecnico recibido conserva su integridad.

#

Pasos

  1. Descargar el archivo suministrado.
  2. Obtener el hash SHA-256 calculado.
  3. Compararlo con el hash entregado oficialmente.
  4. Determinar si coinciden.
  5. Registrar el resultado.
  6. Conservar captura o salida del comando.

Ejemplos tecnicos

Windows: Get-FileHash .\recurso.zip -Algorithm SHA256
Linux: sha256sum recurso.zip

Son ejemplos ilustrativos, no requisitos adicionales.

Datos sensibles

Proteccion de datos sensibles

No basta con identificar un mecanismo de cifrado; debe evaluarse si protege correctamente la informacion.

P

Aspectos a revisar

  • Datos sensibles identificados.
  • Usuarios o procesos autorizados.
  • Controles de acceso asociados.
  • Datos visibles en texto plano.
  • Copias temporales o respaldos sin proteccion.
  • Riesgo de divulgacion.

Ejemplo ficticio

Una base de datos de pruebas contiene campos clinicos simulados en texto plano dentro de una copia de respaldo. La evidencia sugiere ausencia de cifrado en reposo y requiere control criptografico sobre el respaldo.

I

Hallazgo>Evidencia>Riesgo>Mecanismo criptografico>Resultado esperado
